class AttendanceApiController extends BaseController {

        $this->requireRole('student');
        $code = $_POST['pin_code'] ?? '';

Mẫu thiết kế Singleton & Repository

EduManager
Singleton Pattern (Quản lý Database Connection)

Tạo kết nối CSDL duy nhất trên toàn hệ thống trong suốt một vòng đời request. Tránh lãng phí bộ nhớ và quá tải kết nối MySQL khi có hàng trăm sinh viên cùng quét mã QR điểm danh đồng thời.

Repository Pattern (Tầng trừu tượng dữ liệu)

Tách biệt tầng truy vấn SQL khỏi Controller. Khi cần đổi từ cơ sở dữ liệu MySQL sang PostgreSQL hay NoSQL, ta chỉ cần viết lại Repository mà không cần chỉnh sửa logic nghiệp vụ ở Controller.

PHP (Database & Repository) 
// Singleton Database Connection
class Database {
    private static $instance = null;
    private $conn;
    private function __construct() {
        $this->conn = new PDO("mysql:host=127.0.0.1;...", "user", "pass");
    }
    public static function getInstance() {
        if (!self::$instance) {
            self::$instance = new Database();
        }
        return self::$instance->conn;
    }
}

// Repository xử lý dữ liệu
class UserRepository {
    private $db;
    public function __construct() {
        $this->db = Database::getInstance();
    }
    public function findById($id) {
        $stmt = $this->db->prepare("SELECT * FROM users WHERE id = ?");
        $stmt->execute([$id]);
        return $stmt->fetch(PDO::FETCH_ASSOC);
    }
}

Mẫu thiết kế Strategy & Observer

EduManager
Strategy Pattern (Đa dạng hóa Điểm danh)

Định nghĩa các thuật toán điểm danh khác nhau (QR động, Code PIN bảo mật, hoặc tích tay thủ công của giáo viên) dưới dạng các chiến lược (Strategies) riêng biệt, dễ dàng mở rộng thêm các chiến lược mới (ví dụ điểm danh vân tay, khuôn mặt) mà không thay đổi luồng nghiệp vụ xử lý chính.

Observer Pattern (Hệ thống Cảnh báo)

`AlertEngine` đăng ký làm quan sát viên lắng nghe sự kiện thay đổi điểm số tương tác CPI của sinh viên. Khi điểm CPI được tính lại và phát hiện dưới ngưỡng 50, hệ thống lập tức tự động gửi thông báo đến Admin, Giảng viên và Sinh viên.

PHP (Strategy & Observer) 
// Strategy Interface
interface AttendanceStrategy {
    public function verify($studentId, $sessionData, $inputData): bool;
}

// Strategy cụ thể cho QR động
class QrCodeStrategy implements AttendanceStrategy {
    public function verify($studentId, $sessionData, $inputData): bool {
        // So khớp token QR động thời gian thực & khoảng cách GPS
        return hash_equals($sessionData['qr_token'], $inputData['token']);
    }
}

// Observer phát cảnh báo khi thay đổi điểm CPI
class AlertEngine implements CpiObserver {
    public function onCpiChanged($studentId, $courseId, $newCpi) {
        if ($newCpi < 50) {
            $this->createAlert($studentId, $courseId, "Cảnh báo CPI dưới ngưỡng");
        }
    }
}

⚡ Mã QR Code sinh ngẫu nhiên bảo mật

Giảng viên chiếu mã QR trên màn chiếu lớn. Máy chủ tự động sinh token ngẫu nhiên bảo mật cao (16 bytes bằng hàm `random_bytes`), lưu trữ vào database của phiên học. Sinh viên quét mã QR để truyền token này lên server xác thực.

⏱ Mã PIN giới hạn thời gian nhập

Giảng viên cung cấp một mã PIN ngẫu nhiên gồm 6 chữ số được sinh tự động. Sinh viên phải nhanh chóng nhập mã PIN này trên điện thoại của mình để hoàn tất điểm danh trước khi phiên điểm danh hết hạn và đóng lại.

Các cơ chế kiểm soát của hệ thống
  • Giới hạn thời gian (Expiration): Giảng viên cấu hình thời gian hết hạn linh hoạt (ví dụ: 1 phút, 5 phút, 10 phút) để cổng điểm danh tự động đóng lại, ngăn chặn việc quét muộn.
  • Token duy nhất theo ca học: Mã QR và PIN chỉ có hiệu lực cho một ca học cụ thể tại thời điểm được kích hoạt, không thể tái sử dụng cho ca khác.
  • Tính nhất quán cao: Toàn bộ dữ liệu điểm danh được đồng bộ lập tức sang bảng điểm CPI của sinh viên để đánh giá và theo dõi.

Công thức Tính Chỉ số Tương tác CPI

EduManager
Công thức định lượng CPI lớp học

Chỉ số CPI (Class Participation Index) đánh giá thái độ học tập từ nhiều khía cạnh, tránh phụ thuộc vào một cột điểm đơn lẻ:

CPI = (Chuyên cần × 50%) + (Điểm Quiz × 50%) + Điểm Tương tác (Tối đa 10%)

* Chuyên cần: đi học đầy đủ nhận 100 điểm, đi muộn nhận 50đ, vắng học 0đ.
* Điểm Quiz: Quy đổi theo tỷ lệ % kết quả làm bài trắc nghiệm nhanh.
* Điểm Tương tác (Thưởng): Điểm thảo luận cộng trực tiếp vào CPI (tối đa 10%). Khi tổng điểm CPI vượt quá 100, hệ thống tự động giới hạn tối đa là 100. Điểm thưởng này đóng vai trò bù đắp cho phần điểm chuyên cần hoặc quiz bị thiếu hụt của sinh viên.

PHP (Helper calculateCPI) 
// Hàm tính toán chỉ số CPI tự động
public static function calculateCPI($presentCount, $lateCount, $absentCount, $avgQuizScore, $bonusPoints) {
    // 1. Tính điểm chuyên cần (thang điểm 100)
    $totalSessions = $presentCount + $lateCount + $absentCount;
    if ($totalSessions === 0) return 0;
    
    $attendanceScore = (($presentCount * 100) + ($lateCount * 50)) / $totalSessions;
    
    // 2. Tính điểm CPI tổng hợp
    $cpi = ($attendanceScore * 0.5) + ($avgQuizScore * 0.5) + $bonusPoints;
    
    // Khống chế điểm tối đa là 100
    return min(100, max(0, round($cpi, 1)));
}
