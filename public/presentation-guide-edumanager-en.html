Design Patterns

Singleton & Repository Patterns

EduManager
Singleton Pattern (Database Connections)

Ensures that only a single PDO database instance is created and shared per request. This prevents memory leaks and limits maximum active MySQL connections when hundreds of students log in concurrently.

Repository Pattern (Data Abstraction)

Decouples SQL query execution from Controllers. If the database engine changes (e.g., swapping MySQL for PostgreSQL), the developer only needs to rewrite the Repository methods without touching Controller logic.

PHP (Database & Repository) 
// Singleton Database Pattern
class Database {
    private static $instance = null;
    private $conn;
    private function __construct() {
        $this->conn = new PDO("mysql:host=127.0.0.1;...", "user", "pass");
    }
    public static function getInstance() {
        if (!self::$instance) {
            self::$instance = new Database();
        }
        return self::$instance->conn;
    }
}

// Repository data access abstraction
class UserRepository {
    private $db;
    public function __construct() {
        $this->db = Database::getInstance();
    }
    public function findById($id) {
        $stmt = $this->db->prepare("SELECT * FROM users WHERE id = ?");
        $stmt->execute([$id]);
        return $stmt->fetch(PDO::FETCH_ASSOC);
    }
}

Design Patterns

Strategy & Observer Patterns

EduManager
Strategy Pattern (Decoupling Verification)

Encapsulates different verification algorithms (QR scanner, PIN code check, or teacher override) into distinct strategy classes, allowing the code to select the correct strategy dynamically and support new methods (e.g., face recognition) with ease.

Observer Pattern (Alert Dispatching)

`AlertEngine` registers as an observer monitoring changes in student CPI scores. When a student's score drops below the threshold, the observer triggers warnings to students, teachers, and admins.

PHP (Strategy & Observer) 
// Strategy Interface
interface AttendanceStrategy {
    public function verify($studentId, $sessionData, $inputData): bool;
}

// Concrete Strategy for QR Verification
class QrCodeStrategy implements AttendanceStrategy {
    public function verify($studentId, $sessionData, $inputData): bool {
        // Validate secure QR session token
        return hash_equals($sessionData['qr_token'], $inputData['token']);
    }
}

// Observer triggering warnings on CPI change
class AlertEngine implements CpiObserver {
    public function onCpiChanged($studentId, $courseId, $newCpi) {
        if ($newCpi < 50) {
            $this->createAlert($studentId, $courseId, "Low CPI threshold warning");
        }
    }
}

Attendance

Secure Attendance Verification

EduManager
⚡ Secure Random QR Code Generation

Lecturer displays a session-bound QR code. The server dynamically generates a secure, cryptographically random 16-byte token (`random_bytes`). Students scan this QR code to submit and verify the token.

⏱ Time-Limited PIN Entry

Lecturer broadcasts a randomly generated 6-digit PIN code. Students must input this PIN code on their dashboard to complete log-in prior to expiration.

System Control Mechanisms
  • Temporal Expiration: Configure custom scan windows (e.g. 1, 5, or 10 minutes) after which the portal closes automatically to prevent late submissions.
  • Unique Session Tokens: PINs and QR tokens are bound only to the specific active lecture block, preventing reuse.
  • Instant CPI Sync: Attendance records are synced immediately into the engagement engine to update student CPI status.

Algorithm

Real-Time CPI Calculation Formula

EduManager
Defining Classroom Engagement (CPI)

Class Participation Index (CPI) assesses student involvement dynamically through multiple indicators:

CPI = (Attendance × 50%) + (Quiz Score × 50%) + Bonus Interaction (Max 10%)

* Attendance: 100 points for present, 50 points for late, 0 points for absent.
* Quiz Score: Normalized percentage score calculated from online mini-quizzes.
* Bonus Interaction Points: Forums replies scored by the lecturer. Added directly to CPI (capped at 100). Serves as a buffer to recover from missed classes or poor quiz performance.

PHP (Helper calculateCPI) 
// Automated student CPI calculation
public static function calculateCPI($presentCount, $lateCount, $absentCount, $avgQuizScore, $bonusPoints) {
    // 1. Calculate Attendance Score (scale 100)
    $totalSessions = $presentCount + $lateCount + $absentCount;
    if ($totalSessions === 0) return 0;
    
    $attendanceScore = (($presentCount * 100) + ($lateCount * 50)) / $totalSessions;
    
    // 2. Compute aggregate CPI
    $cpi = ($attendanceScore * 0.5) + ($avgQuizScore * 0.5) + $bonusPoints;
    
    // Cap maximum score at 100
    return min(100, max(0, round($cpi, 1)));
}
